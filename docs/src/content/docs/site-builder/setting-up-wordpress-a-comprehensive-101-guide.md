**Setting up WordPress: A Comprehensive 101 Guide**

WordPress is an open-source website creation platform that integrates with . If you run WordPress websites, you can manage them all in one spot from the dashboard. With the search and replace feature, will check for all URLs with HTTP and HTTPS and update them in the new domain.

### **FAQs**




**Question:** I'm unable to add the same domain to my other account/location:
**Answer:** The same domain/subdomain can't be used in 2 locations




**Question:** Why can't I update my primary domain?
**Answer:** The primary domain can be updated if you have added an additional domain.





**Question:** I've added the A / CNAME record, but the domain is still not being added
**Answer:** This can happen for a few reasons.

-   There is a typo in your domain name.  
    In this scenario, fixing the typo will resolve your issue
-   Your DNS changes haven't been propagated yet.  
    In this case, you must wait longer and try again after a few hours or the next day to see if it works.
-   Your DNS configuration is not set up correctly.  
    Please contact your Domain Provider and discuss the errors with their support team.
-   Maybe you have conflicting (multiple) records for the same subdomain \[_for example, if blog.mydomain.com has a CNAME record pointing to wp3.msgsndr.com but it also has an A record pointing to some other provider_\]  
    In such cases, removing the other duplicate record will fix your issue.




**Question:** I can't connect my root domain (ex., mydomain.com)
**Answer:** Please make sure there are no additional AAA, TXT, records, etc if you plan to use that slug/root domain for WordPress




**Question:** How many WordPress sites can I have per account?
**Answer:** Only one site per account at the moment.




**Question:** How can I import an existing WordPress site into ?
**Answer:** To import an existing WordPress site, use the All-in-One WP Migration plugin. Install the plugin on your current WordPress site, export the data, and import it into your new WordPress setup on .




**Question:** What should I do if my WordPress site breaks?
**Answer:** If your WordPress site breaks, you can use the Backup & Restore feature in the WordPress Dashboard on . Daily backups are performed at 05:00 AM CST and stored for the past thirty days, allowing you to restore your site to its previous state if needed.




**Question:** What is the purpose of SSH access?
**Answer:** SSH access allows secure, command-line management of your server, ideal for advanced users and developers needing more control over site operations.




**Question:** Do I need a password to log in?
**Answer:** No, SSH authentication uses public key authentication rather than passwords for enhanced security.




**Question:** Can I use SSH for all types of tasks?
**Answer:** Yes, SSH access allows you to manage plugins, clear caches, import large files, and more directly on the  




**Question:** Can I switch the server location after setting up my WordPress site?
**Answer:** No, once a server location is selected during the WordPress site setup, it cannot be changed. The selected server location is permanent and optimized for faster content delivery and reduced latency.


**Question:How do I delete a WordPress site?
**Answer:** Use the **Delete Site** option in advanced settings. This will remove all files, databases, and domains associated with the site.  




**Question:** Can I update multiple WordPress sites at once?

**Answer:** Yes. From the main WordPress page, you can perform bulk actions across multiple installs. This includes updating **plugins**, **themes**, and **PHP versions**, as well as **clearing cache** settings for all selected sites simultaneously to save time.




**Question:** How can I prevent search engines from indexing my website?

**Answer:** You can enable **SEO Indexing** in the Advanced Settings. This adds a robots.txt file to your site that instructs search engines and bots not to crawl or index your pages, giving you full control over your site’s privacy.
