How to Edit User Information and Permissions  

User Permissions allow you to grant or restrict certain access and capabilities for different team members. Follow the steps below:  

-   Navigate to Settings > My Staff to access Team Management.
-   Choose the User you want to edit (information or permissions). Click **Edit**.

![](https://assets2.modalsupport.net/1758903483117-01.png)

-   To update basic **User Info** such as name, email, phone number, password, email signature, and calendar assignment, go to the User Info and make your changes.

![](https://assets2.modalsupport.net/1758903596315-02.png)

-   To update user permissions and roles, click on the **Roles & Permissions** option and toggle on/off which settings you would like to update.

![](https://assets2.modalsupport.net/1758903683280-03.png)

**NOTE:** If you are not the main/primary user (as an administrator) for , you will not be able to make changes through this option.  
**  
User Management Permissions**  

Administrators can control who can create, edit, and delete users, as well as who can only view the user list.  

Administrators can assign permissions: **"View and Manage Users"** for full access or **"View Users"** for read-only access. If both permissions are disabled, the Team and MyStaff sections will be hidden from the side navigation bar.

**Additional Permissions:**  

**Only Assigned Data:**  

"Only Assigned Data" is a great way to limit salespeople from seeing each other's leads, opportunities, and conversations

![](https://assets2.modalsupport.net/1758905226119-06.png)

Limiting user access to only assigned data in the subaccount will constrain their activity and capabilities.  
This implies that users will exclusively view or have access to the information they own.

-   Contacts: Users will see only their assigned contacts and the corresponding conversations.
-   Campaigns (view-only): Users can view the campaign and its content, but won't have the ability to edit it.
-   Workflows (View-only): If turned on, users can view the workflow and its content, but won't have the ability to edit it.

![](https://assets2.modalsupport.net/1758905536500-07.png)

-   Calendar: Users will only observe their assigned calendar and appointed meetings. Users can have specific View and Manage permissions for Calendars, Appointments, and Groups. Access can be restricted to only assigned data, and users can be given independent manage permissions to control what they can create, edit, or delete.

![](https://assets2.modalsupport.net/1758905604821-08.png)

![](https://assets2.modalsupport.net/1758905675896-09.png)

-   Pipeline/Opportunities: Users will exclusively access their assigned opportunities in the pipeline.**  

    Note:** There is no option to assign tags, funnels, workflows, etc. Therefore, toggling the "only assigned data" feature will restrict user access to these functionalities.  
    **  
    User permissions for exporting payment data as a CSV**  

    Admins can configure user permissions to export payment data as CSV from the account. This applies to exporting invoices, transactions, or subscriptions to CSV. This feature is useful when businesses do not want specific users to have export permissions on their payment data. The toggle is turned on by default for all users, which means that they will have permission to export data by default.

![](https://assets2.modalsupport.net/1758905828924-10.png)


**Granular Payment Permissions**  

This introduces detailed payment-related permissions, splitting them into six specific sections: Orders, Subscriptions, Transactions, Taxes, Products, and Payment Settings. This allows businesses to assign precise access for each section, security and control over sensitive financial actions.  
**  
How to Use:**  
Go to Settings → My Staff → 'Staff Member' Edit → Roles & Permissions to configure permissions for each section.

**FAQs**




**Question:** How can I edit a user's information or permissions?
**Answer:** Navigate to **Settings > My Staff**, select the user, and click **Edit**. Use the **User Info** tab to update details like name, email, and calendar assignments or the **Roles & Permissions** tab to modify access and roles.




**Question:** What is the "Only Assigned Data" feature?
**Answer:** This feature restricts users to view and manage only the data assigned to them, including contacts, calendars, and pipeline opportunities. It also limits access to editing campaigns and workflows.


**Question:What happens if both "View and Manage Users" and "View Users" permissions are disabled?
**Answer:** If both the "View and Manage Users" and "View Users" permissions are disabled, the Team and MyStaff sections will no longer be visible in the side navigation bar.




**Question:** Can I restrict a user’s ability to export payment data?
**Answer:** Yes, admins can toggle permissions to restrict exporting invoices, transactions, and subscriptions to CSV. By default, this permission is enabled for all users.




**Question:** How do I assign a phone number or voicemail to a user?
**Answer:** Under **Call and Voicemail Settings**, assign a single phone number and upload a voicemail recording for the user.




**Question:** Can I restrict users from viewing certain calendars or appointments?
**Answer:** Yes, with granular permissions, you can restrict users to only view and manage the calendars, appointments, and groups they are assigned to, ensuring they only have access to relevant data.  
**  
Question:How can I configure granular payment-related permissions for users?**
**Answer:** Admins can configure granular payment-related permissions by navigating to Settings → My Staff → 'Staff Member' Edit → Roles & Permissions. There, permissions for specific payment sections such as Orders, Subscriptions, Transactions, Taxes, Products, and Payment Settings can be assigned, allowing for precise control over user access to sensitive financial data.




**Question:** What permissions can be assigned to limit user access to only specific types of data?
**Answer:** Admins can enable the "Only Assigned Data" feature to limit users to only see and manage the data assigned to them. This includes restricting access to contacts, campaigns, workflows, calendars, appointments, pipeline opportunities, and payment data.




**Question:** Why can't I see the "Edit" button for one of my team members?
**Answer:** Only users with **Administrator** status can edit other users. If you are an "User" level account, you will not have the authority to change the permissions of your peers or superiors.




**Question:** Can a user manage their own calendar if they are restricted to assigned data?

**Answer:** Yes. You can provide **independent manage permissions** for Calendars and Appointments. This allows a user to create and edit their own meetings while still being restricted from viewing other team members' calendars.




**Question:** If I enable "Only Assigned Data," can a salesperson still see my master contact list?

**Answer:** No. Enabling "Only Assigned Data" creates a "siloed" experience. That user will only be able to see and search for contacts that have been explicitly assigned to them. They will not be able to see, email, or call leads assigned to other team members or leads that currently have no assigned owner.
